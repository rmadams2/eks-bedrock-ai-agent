# AWS EKS AI Agent with Bedrock & New Relic Observability

This repository contains a production-ready setup for deploying a Python FastAPI AI Agent to Amazon EKS. 

It leverages **Amazon Bedrock** (via Cross-Region Inference Profiles) for AI generation, uses **EKS Pod Identity** for secure, keyless IAM authentication, exposes the API via an **AWS Network Load Balancer (NLB)**, and automates deployment and monitoring using **Terraform** and **New Relic**.

## Architecture

* **Application:** Python FastAPI containerized with Docker, instrumented with the New Relic APM agent.
* **Authentication:** EKS Pod Identity grants the Kubernetes pods direct IAM permissions to invoke Bedrock without embedding long-lived AWS credentials.
* **Inference:** AWS Bedrock Cross-Region Inference Profiles (`us.anthropic.claude-3-5-sonnet-20241022-v2:0`).
* **Networking:** AWS Network Load Balancer (NLB) exposing the agent to the internet.
* **Observability:** New Relic Scripted API Synthetic Monitors perform continuous evaluation and trigger NRQL alerts for security anomalies (e.g., prompt injections).

## 📋 Prerequisites

Before deploying, ensure you have the following installed and configured on your local machine:

1. **AWS CLI (v2):** Configured with administrator or proper deployment permissions (`aws configure`).
2. **Docker:** Installed with `buildx` enabled for multi-architecture builds.
3. **kubectl:** Installed and configured to communicate with your target EKS cluster.
4. **Terraform:** Version 1.5.0 or higher.
5. **Amazon Bedrock Access:** You must explicitly request and enable model access for Anthropic Claude 3.5 Sonnet and/or Amazon Nova Pro in the AWS Console for your target region (e.g., `us-west-2`).
6. **New Relic Account:** An active account. You will need your **Ingest License Key** and a **User API Key**.

## 🚀 Step-by-Step Implementation

### Step 1: Clone the Repository
Clone this repository and navigate into the project directory:
```bash
git clone [https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git](https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git)
cd YOUR_REPO_NAME
```

### Step 2: Build and Push the Docker Image
Navigate to the application directory and build the multi-architecture Docker container. Push it to your preferred container registry (e.g., Amazon ECR, Docker Hub).

```bash
cd app/

# Replace with your actual registry URI
docker buildx build \
  --platform linux/amd64,linux/arm64 \
  -t your-registry/eks-bedrock-agent:v1 \
  --push .

cd ..
```

### Step 3: Configure Terraform Variables
Navigate to the Terraform directory and set up your environment variables.

```bash
cd terraform/
cp terraform.tfvars.example terraform.tfvars
```

Open `terraform.tfvars` in a text editor and fill in your specific infrastructure and observability details:
```hcl
aws_region           = "us-west-2"
eks_cluster_name     = "your-existing-eks-cluster-name"
app_image            = "your-registry/eks-bedrock-agent:v1"
newrelic_account_id  = "YOUR_NEW_RELIC_ACCOUNT_ID"
newrelic_license_key = "YOUR_NEW_RELIC_LICENSE_KEY"
newrelic_api_key     = "NRAK-YOUR_NEW_RELIC_USER_API_KEY"
```

### Step 4: Deploy the Infrastructure
Initialize Terraform to download the required providers (AWS, Kubernetes, New Relic), review the execution plan, and apply the configuration.

```bash
# 1. Initialize Terraform
terraform init

# 2. Preview the changes
terraform plan

# 3. Apply the infrastructure
terraform apply
```
Type `yes` when prompted. Terraform will configure your IAM roles, EKS Pod Identity, Kubernetes Deployments/Services, and New Relic Synthetics. 

Once complete, Terraform will output your public load balancer URL:
```text
Outputs:

load_balancer_hostname = "a1b2c3d4...elb.us-west-2.amazonaws.com"
```

## 🧪 Verification & Testing

### 1. Test the API Endpoint
Wait 2-3 minutes for AWS to finish provisioning the Network Load Balancer hardware, then send a test request using the hostname output by Terraform:

```bash
curl -X POST http://<YOUR_LOAD_BALANCER_HOSTNAME>/generate \
  -H "Content-Type: application/json" \
  -d '{"prompt": "Explain Kubernetes in one sentence.", "user_id": "test_user_1"}'
```
*If successful, you will receive a JSON response containing the text generated by AWS Bedrock.*

### 2. Verify Observability in New Relic
* **APM & Services:** Log into New Relic and navigate to APM to view latency, throughput, and token count metrics for your `EKS-Bedrock-Agent`.
* **Synthetic Monitoring:** Navigate to Synthetics to view the execution logs of the automated evaluator script Terraform deployed.
* **Alerts & AI:** View the deployed `AI Agent Security Policy` which will automatically trigger a critical violation if a user attempts a prompt injection.

## 🧹 Cleanup

To prevent ongoing cloud charges, you can tear down all resources managed by Terraform by running:

```bash
cd terraform/
terraform destroy
```
Type `yes` when prompted.
